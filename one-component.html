One component

There's only one React component on this page, which means that while we're still using it to project state, don't expect too much interactivity. Nothing else on this page should react to any changes made by the component.

Also note the injected initial state from what could be server variables. This is key when using Razor to create initial state where the page then uses JS to take over rendering/interactivity.
